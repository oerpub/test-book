Undo Expectations 

1. Basically what I expect to happen when I hit undo, is that things undo up to the last structural change. If I hit undo anytime in this paragraph, some subset of what I typed should disappear, but not the heading above me or the paragraph above (if there were one) or the semantic element above me.

2. Inside this subheading hitting undo should only delete some part of this heading, not the para before. 

4. Hitting undo while in the title, should only delete up to the title.

3. Right after I add this note, undo should just remove the note. If I hit undo after typing some in this para inside the note, only characters inside this para should disappear and maybe this whole para, but not more.

5. If I delete this note and hit undo, the entire note should restore, and nothing else should happen.

6. If I hit undo and then redo, my document should look like it did just before I hit undo.

7. If I select "Copy quotation" and then hit undo, it should undo something I did before that action. Since copy doesn't change anything.

8. If I actually copy the quotation and then paste the quote, then undo should undo the paste, and that is it.

9. If I am typing away and then I bold some text and then hit undo, right after bolding the text, the text should unbold and nothing else should undo. The same for other operations like that (italics, underline, subscript, superscript).

10. If I make a link out of some text, and then hit undo before doing anything else, the link should disappear, but the text under the link stay. If I add a link at the cursor, and then hit undo, the link and the text should disappear.

11. If I add a list

  • And then type some and hit undo, the undo should go at maximum to the list bullet.
  • If I add a bullet but no text and then hit undo, the bullet should disappear.

 12. If I add an image, table, video, math equation and then immediately hit undo, it should disappear, but make sure that redo brings it back with all of its associated metadata if any.

13. If I change some text to a header and then hit undo, the text should change back to normal.

14. Math! Undo should not cross math borders.  x 2 x^2 If I undo after typing chars after math, math shouldn't be undone. Then math should be undone, but should completely redo also. Then chars before the math.
